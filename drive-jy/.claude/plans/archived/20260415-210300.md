# 초기 커밋 — 프로젝트 베이스라인 확립

## 작업 목표

지금까지 작성된 모든 파일을 git 저장소의 첫 커밋으로 기록한다.
- Phase 1 구현 (사용자 인증, 보안 설정, 권한 프레임워크)
- 테스트 하네스 (IntegrationTestBase, FakeUserRepository, 17개 테스트)
- Claude Code 워크플로우 (hook 6종, agent 5종, CLAUDE.md 가이드)
- 커밋 컨벤션 가이드

## 변경할 파일

전체 프로젝트 파일을 단일 초기 커밋으로 처리. 이후 커밋부터 Angular Convention 으로 세분화.

제외:
- `bash.exe.stackdump` 삭제 (Cygwin 충돌 잔여물)
- `.claude/settings.local.json` (이미 .gitignore)
- `.claude/state/current-plan.md` (이미 .gitignore)

## 완료 조건

- `git commit` 성공 (Angular Convention 형식)
- `git log` 로 커밋 확인
- `post-commit-clear.sh` hook이 계획서를 archived/ 로 자동 이동
