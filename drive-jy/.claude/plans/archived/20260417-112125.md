E2E 테스트 7개 실패 수정 (auth 6개 통과, 7개 실패)

## 실패 원인 및 수정 파일

### 1. SecurityConfig.kt — CookieCsrfTokenRepository 추가
- 원인: CSRF 토큰이 세션에만 저장됨. fetch API에서 XSRF-TOKEN 쿠키 읽기 불가 → 모든 API POST 403
- 수정: `.csrf { it.csrfTokenRepository(CookieCsrfTokenRepository.withHttpOnlyFalse()) }`
- 파일: src/main/kotlin/demo/drive/common/config/SecurityConfig.kt

### 2. e2e/specs/auth/auth.spec.ts — 로그아웃 셀렉터 수정
- 원인: `button[formaction="/auth/logout"]` 매칭 없음. 실제는 form 안의 일반 submit 버튼
- 수정: `page.click('form button:has-text("로그아웃")')`

### 3. e2e/specs/file/file.spec.ts — 두 곳 수정
- "내 드라이브" strict mode: `page.locator('h1').filter({ hasText: '내 드라이브' })` 사용
- 권한 없는 접근 404: 기대 배열에 404 추가 `[302, 401, 403, 404]`
