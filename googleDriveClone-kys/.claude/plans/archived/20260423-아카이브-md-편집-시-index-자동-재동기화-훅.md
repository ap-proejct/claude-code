# 아카이브 md 편집 시 INDEX 자동 재동기화 훅 (Layer A)

## Context

현재 `INDEX.jsonl` 은 분석 커맨드(`/harness-trend`, `/antipattern-scan`, `similar-plans.sh`) 실행 시점에 `plan-resync.sh` 로 재생성된다. 사용자가 IDE에서 아카이브 md 의 `score` 를 직접 고치는 경우는 이걸로 충분히 커버된다 (다음 분석 실행 때 반영).

하지만 **Claude가 아카이브 md 를 Edit/Write 하는 경우** (예: 사용자가 "score 4로 바꿔줘" 요청) 에는 파일은 갱신됐는데 INDEX 는 낡은 상태로 남는 구간이 생긴다. 이 순간에 사용자가 `cat INDEX.jsonl` 하거나 다른 도구가 INDEX 를 참조하면 부정합이 보인다.

훅은 **Claude의 tool 호출에만 반응**하므로 사용자 IDE 편집까지는 잡을 수 없지만, **Claude 편집 케이스는 완벽히 잡아낼 수 있다**. 훅 파일 하나와 settings.json 한 줄로 "Claude가 건드린 archive md 는 즉시 INDEX 동기" 가 보장된다.

## 설계

### 신규 훅: `.claude/hooks/resync-on-archive-edit.sh`

**트리거**: PostToolUse(Edit|Write)

**동작**:
1. stdin JSON 에서 `tool_input.file_path` 추출
2. 경로가 `.claude/plans/archived/*.md` 패턴이면 (INDEX.jsonl 제외) `bash .claude/scripts/plan-resync.sh` 호출
3. 그 외 경로면 조용히 exit 0

**중요 예외**:
- **INDEX.jsonl 자체는 제외** — plan-resync 가 INDEX 를 Write 하면 훅이 재발동하지만, Claude 가 INDEX 를 직접 편집하는 경우는 없으므로 파일명 기반 필터로 안전. 그래도 이중 안전장치로 `INDEX.jsonl` 포함 시 skip.
- **archived 가 아닌 plans/*.md 는 제외** — 활성 플랜은 `track-active-plan.sh` 가 관리. 여기서 건들지 않음.

**구현 스케치**:
```bash
#!/usr/bin/env bash
set -u

project_dir="${CLAUDE_PROJECT_DIR:-$(pwd)}"

file=$(python3 -c '
import sys, json
try:
    d = json.load(sys.stdin)
    print(d.get("tool_input", {}).get("file_path", ""))
except Exception:
    pass
' 2>/dev/null)

[[ -z "$file" ]] && exit 0

case "$file" in
  *.claude/plans/archived/*.md|.claude/plans/archived/*.md)
    # INDEX.jsonl 자체는 건들지 않음 (무한 루프 방지 이중 안전장치)
    case "$(basename "$file")" in
      INDEX.jsonl) exit 0 ;;
    esac
    bash "$project_dir/.claude/scripts/plan-resync.sh" 2>/dev/null || true
    ;;
esac

exit 0
```

```json
  ]
}
```

`track-active-plan.sh` 와 나란히 위치. 순서 무관(독립).

## 검증

1. **Claude 편집 시 INDEX 동기 확인**:
   - 기존 `archived/20260423-*.md` 의 `score: 4` 를 Edit 으로 `score: 5` 로 수정
   - 훅 실행 직후 `cat .claude/plans/archived/INDEX.jsonl` → `"score": 5` 로 즉시 반영돼야 함

2. **무관한 파일 편집 시 no-op 확인**:
   - 아무 `.claude/hooks/*.sh` 를 Edit → plan-resync 실행 흔적 없어야 함 (INDEX mtime 변화 없음)

3. **활성 플랜 편집 시 no-op 확인**:
   - `/home/rhkdb/.claude/plans/*.md` (활성 플랜) Edit → `track-active-plan.sh` 만 돌고 resync 훅은 no-op

4. **INDEX.jsonl 자체 제외 확인**:
   - 훅 자체가 INDEX.jsonl 을 write 해도 (plan-resync 통해), 그 write 이벤트에서 훅이 재발동하지 않음 (Claude 가 직접 INDEX 쓰는 일이 없으니 이론적 시나리오지만 안전망 확인)
