# D 블록 로드맵 (1-D → 1-B 순차)

## Context

D 블록 착수 단계. 처음 후보 4개(1-A 로그인 UX / 1-B 업로드 진행률 / 1-C 휴지통 / 1-D 파일명 truncate) 중 코드 조사 결과:

- **1-A**: 이미 인라인 빨간 텍스트로 구현됨 → 사용자 판단으로 **스킵**
- **1-C**: 복원·영구삭제·휴지통 비우기 모두 프론트·백엔드 완전 구현 → 사용자 판단으로 **스킵**
- **1-D**: `FileItem.jsx` 에 이미 `truncate` CSS 적용돼 있으나 **`title` 속성 누락** — 마우스 오버 시 전체 파일명 보이지 않음 (폴더명·파일명 모두)
- **1-B**: `UploadButton.jsx` 가 `onUploadProgress` 콜백 미구현. `setUploading(true/false)` 로 버튼 상태만 토글, 진행률 UI 없음

남은 2개 항목을 **각각 별도 Plan 사이클** 로 진행. 본 플랜은 로드맵 용도 — 여기서 실제 코드 변경은 하지 않고, 각 하위 플랜의 스코프와 순서만 정의.

## 1-D. 파일명 tooltip (첫 번째 사이클)

### 목표
마우스 오버 시 잘린 전체 파일명이 보이도록 `title` 속성 추가.

### 스코프
- **수정 파일**: `frontend/src/components/FileItem.jsx` (그리드 뷰 48-50, 리스트 뷰 86 근처)
- 기존 `title` 속성(파일 행 컨테이너의 "폴더 열기"/"클릭하여 다운로드" 액션 설명) 을 덮지 않으면서 **파일명 `<p>` 에 별도 `title={name}`** 추가
- 폴더·파일 모두 대상

### 검증
- Playwright MCP Flow 1 로 로그인 → 파일 목록에서 임의 파일 hover → `browser_snapshot` 으로 title 속성 존재 확인 (접근성 트리에 노출되는지)
- 또는 실제 브라우저에서 시각 확인 (스크린샷 1컷)

### 예상 규모
30분 이내. 한 두 줄 추가가 전부.

## 1-B. 업로드 진행률 (두 번째 사이클)

### 목표
업로드 중 0~100% 진행률이 시각적으로 드러나도록 UI 추가.

### 스코프 (대략 — 자체 플랜 사이클에서 확정)
- **주요 파일**:
  - `frontend/src/components/UploadButton.jsx` — axios 호출에 `onUploadProgress` 콜백 추가
  - `frontend/src/pages/DrivePage.jsx:55-74` — 드래그 업로드도 동일 적용
  - progress bar 컴포넌트 신규 or 인라인 Tailwind 구현
- **백엔드**: 변경 없음 (multipart 경로 그대로)
- `uploading` boolean → `uploadProgress: null | number(0-100)` 로 확장

### 검증
- Flow 2 재실행 — 진행률 바가 렌더링되고 완료 시 목록 반영 확인
- `browser_take_screenshot` 으로 진행 중 상태 1컷

### 예상 규모
2~4시간. 코드량은 적으나 UX 디테일(애니메이션·0/100 경계 처리·에러 시 롤백) 에 시간 소요.

## 실행 방침

- **각 항목마다 Plan 모드 재진입** — 본 로드맵은 순서·스코프 참고용이고, 실제 구현은 하위 플랜에서
