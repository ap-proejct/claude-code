# Flow 1·2·3 전체 통과 — WSL 네이티브 스택

Playwright MCP 2차 실행. 프론트/백엔드/브라우저 모두 WSL 에서 기동해 proxy 이슈 우회 성공.

## 환경 전환

- 이전(2026-04-23): frontend·backend(Windows) + browser(WSL). Vite proxy 매칭 실패로 Flow 1 차단
- 이번: 전부 WSL. `localhost` 가 동일 네트워크 공간이라 proxy 없이 바로 통신. 기존 proxy 설정 그대로 두고도 정상 동작

## 선행 작업

- **비밀번호 재설정**: DB 의 `test@test.com` 해시가 `test` 와 불일치. CLAUDE.md 기록(`test/test`) 은 스테일
  - `POST /api/auth/reset-password` 에 보안 답변 `test` + 새 비번 `testtest1` (8자+) 전송해 재설정
- **업로드 파일 경로**: Playwright MCP 는 `.playwright-mcp/` 와 프로젝트 루트 하위만 허용. `/tmp` 직접 업로드 시 `File access denied` → 프로젝트 디렉토리로 복사 후 업로드
