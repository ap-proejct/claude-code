# 캡처 도구 나머지 3종 검증 (snapshot·console·network)

스크린샷 검증(플랜 `20260424-playwright-mcp-스크린샷-캡처-동작-검증`) 에 이어 증거 캡처 규칙의 나머지 3개 도구 실동작 확인.

## 결과 요약

| 도구 | `filename` 인자 | 파일 저장 | 경로 제약 | 비고 |
|------|----------------|----------|----------|------|
| `browser_snapshot` | ✅ 있음 | ✅ md | 프로젝트 루트 하위 OK | yaml-like 접근성 트리 |
| `browser_console_messages` | ✅ 있음 | ✅ txt | 동일 | `level` 필수, `all:true` 로 세션 전체 |
| `browser_network_requests` | ✅ 있음 | ✅ text(확장자 무시) | 동일 | `static:false` 면 정적 리소스 필터됨 |

**`Write` 툴로 2단계 저장할 필요 없음** — 세 도구 모두 `filename` 인자 내장. 플랜 리스크 섹션에 적었던 우려(네트워크·콘솔은 응답만 주고 파일 저장은 우리 몫일 가능성) 은 해소.

## 도구별 관찰

### `browser_snapshot(filename)`

- 접근성 트리가 yaml-ish markdown 으로 저장 (`- generic [ref=e4]:` 형태)
- 크기: 로그인 페이지 699 B
- `ref=eN` 참조는 직후 호출의 `click`/`type` 에 그대로 사용 가능 — 저장본에서 복사해 다시 조작 가능
- 용도: DOM 구조 증거 (배지 누락, role 오용, 텍스트 diff)

### `browser_console_messages(level, filename, all?)`

- `level`: `error|warning|info|debug` — "이상 포함 그 이상" 로직
- `all:true` 는 세션 전체, `false`(기본) 는 직전 `browser_navigate` 이후만
- 포맷: 텍스트, 첫 줄에 요약(`Total messages: 4 (Errors: 0, Warnings: 0)`), 이후 `[INFO]`/`[ERROR]` prefix 라인
- 크기: 로그인 1회 네비게이션 2,284 B (13 메시지)
- 발견: 이전 세션 잔재(502 에러, `vite server connection lost`) 가 `all:true` 에 포함됨 → "순수 이번 세션만" 을 원하면 `all` 생략

### `browser_network_requests(static, requestBody, requestHeaders, filename, filter?)`

- 포맷: **JSON 이 아닌 텍스트** (확장자를 `.json` 으로 줬어도 사람이 읽기 편한 라인 포맷으로 저장됨)
  ```
  [POST] http://localhost:5173/api/auth/login => [502] Bad Gateway
    Request body: {"email":"test@test.com","password":"testtest1"}
  ```
- `static:false` + 정적 페이지(로그인) = **0 바이트** 결과. 의미있는 증거를 얻으려면 실제 API 호출을 먼저 트리거해야 함
- `filter: '/api/'` 같은 regexp 로 관심 요청만 추릴 수 있음
- `requestBody:true` 가 **비밀번호도 plaintext 로 기록** → 민감정보 포함 시 저장 경로·공유 주의. 평소엔 `false` 유지하고 꼭 필요할 때만 킬 것

## 증거 캡처 규칙 업데이트 제안 (e2e-flows.md 반영 여부는 별도 판단)

- **파일명 규칙**: 3개 도구 모두 `.md`/`.txt`/`.json` 확장자 가능하나 network 는 JSON 이 아닌 텍스트라 `.txt` 가 더 정확. 일관성 위해 네트워크는 `.txt` 로 갈지 `.json` 유지할지 컨벤션 결정 필요
- **console `level`**: 버그 탐지용이면 `warning` 기본값 권장 (info 는 노이즈 과다)
- **network 주의**: `requestBody:true` 는 인증 flow 캡처 시 비번 누출 위험. 저장한 `login-attempt-network.json` 이 실제 샘플. `.gitignore` 로 보호하거나 커밋 전 스크러빙 필요
