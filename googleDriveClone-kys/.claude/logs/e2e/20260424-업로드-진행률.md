# 업로드 진행률 기능 검증

플랜 `업로드-진행률-표시` 구현 후 Playwright MCP 로 동작 확인.

## 실행 요약

- 5MB 파일 업로드 → 201 Created, 목록 반영
- 100MB 파일 업로드 → 201 Created, 목록 반영
- 업로드 후 버튼 상태 `파일 업로드` 로 정상 복귀 (uploading=false)

## 제약: 진행률 중간 캡처 불가

Playwright MCP 는 `browser_file_upload` 가 업로드 완료까지 블로킹. 따라서 "업로드 중 NN%" 중간 상태를 스크린샷/스냅샷으로 직접 포착하지 못함. 코드 경로·완료 결과로 기능 정상 동작을 간접 검증.

후속 방법(필요 시):
- `browser_run_code` 로 async 업로드 시작 + setTimeout 후 스크린샷 합성 스크립트 작성
- CDP 네트워크 throttle 걸어 업로드를 일부러 지연시켜 포착

## 증거

- `upload-completed-list.png` — 업로드 완료 후 목록에 5MB·100MB 파일 노출
- `upload-network.json` — `[POST] /api/files/upload => [201] Created` 2건

## 드래그 업로드 경로

코드 변경은 했으나 드래그&드롭 이벤트 시뮬레이션은 Playwright 로 제약이 있어 이번 검증에서는 클릭 업로드만 확인. 드래그 경로는 코드 리뷰 수준에서 같은 `onUploadProgress` 패턴 적용됐음.
