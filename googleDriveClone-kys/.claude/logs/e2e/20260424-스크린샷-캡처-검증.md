# Playwright MCP 스크린샷 캡처 동작 검증

`e2e-flows.md` "증거 캡처(수동 트리거)" 섹션 도입 후 `browser_take_screenshot(filename)` 이 실제로 동작하는지, 저장 경로가 `.claude/logs/e2e/<basename>/` 로 지정 가능한지 검증.

## 실행

1. 프론트엔드만 WSL 기동 (로그인 화면 1컷만 필요하므로 백엔드·MySQL 생략)
2. `browser_navigate('http://localhost:5173/login')`
3. `browser_take_screenshot(type: 'png', filename: '/mnt/c/.../.claude/logs/e2e/20260424-스크린샷-캡처-검증/login-page.png')`
4. `file` + `stat` 로 시그니처·크기 검증

## 결과

- ✅ 지정 경로에 PNG 생성됨: `20260424-스크린샷-캡처-검증/login-page.png`
- ✅ `file` 판정: `PNG image data, 1280 x 720, 8-bit/color RGB, non-interlaced`
- ✅ 크기: 24,887 bytes (의미 있는 시각 정보 담김)

## 관찰

- **절대경로 → 상대경로 자동 축약**: 입력은 절대경로(`/mnt/c/.../login-page.png`)였으나 MCP 응답에서는 프로젝트 루트 기준 상대경로(`.claude/logs/e2e/.../login-page.png`)로 축약돼 표시됨. 실제 파일은 의도한 절대경로에 저장
- **디렉토리 자동 생성**: `mkdir -p` 를 미리 했지만, 기존 `browser_file_upload` 와 달리 screenshot 은 경로 제약이 느슨해 보임 (프로젝트 루트 하위면 OK)
- **한글 디렉토리명 동작**: `20260424-스크린샷-캡처-검증` 같은 한글 경로도 문제없이 저장·조회됨

