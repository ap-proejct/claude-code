# Flow 1 최초 실행 — Vite proxy 막힘

Playwright MCP 붙이고 첫 실행. 네트워크 토폴로지 이슈로 Flow 1 부터 막혀 후속 Flow 미실행.

## 환경 구성 이슈 (해결됨)

- chromium 바이너리 + 시스템 의존성(libnspr4 등) 설치
  - `npx playwright install chromium`
  - `sudo env PATH=$PATH npx playwright install-deps chromium` (nvm 관리 node 는 sudo PATH 미포함)
- `@playwright/mcp` 기본 `--browser chrome` 채널은 WSL 에 Google Chrome 이 없으면 실패. `.mcp.json` 에 `--executable-path` 로 번들 chromium 명시 필요

## Flow 1 차단 원인 (미해결)

axios `baseURL: 'http://localhost:8080'` 하드코딩 → WSL 브라우저에서 `ERR_CONNECTION_REFUSED`.
- WSL chromium 의 `localhost` = WSL 내부. 백엔드는 Windows 에서 기동 중이라 다른 네트워크 공간
- `baseURL` 을 `''` 로 비우고 Vite proxy 도입 시도했으나 **proxy 가 실제 요청에 매칭되지 않는 증상**
  - `GET /api/x` → SPA fallback HTML 200 반환
  - `POST /api/*` → `Vary: Origin` 헤더의 빈 404 (Vite 자체 응답)
- shortcut 형식 · `{ target, changeOrigin, secure }` 객체 형식 · `.vite` 캐시 초기화 · 재시작 모두 시도했으나 동일
